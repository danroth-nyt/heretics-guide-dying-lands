# Contributing to HERETIC MAP GENERATOR

Thank you for your interest in improving this project!

## Development Setup

1. Fork the repository
2. Clone your fork
3. Install dependencies: `npm install`
4. Start dev server: `npm run dev`
5. Make your changes
6. Test thoroughly
7. Submit a pull request

## Code Style

- TypeScript for all source files
- Functional React components with hooks
- Tailwind CSS for styling (custom classes in index.css)
- ESLint rules must pass
- Keep the MÖRK BORG aesthetic

## Areas for Contribution

### High Priority

- **Complete PDF Data**: Fill in all location tables from the source material
- **Mobile Responsiveness**: Improve layout for smaller screens
- **Map Persistence**: Save/load maps to localStorage or JSON

### Medium Priority

- **Additional Territories**: Expand beyond the core four
- **Custom Location Types**: Allow users to add their own
- **Enhanced Roads**: More visual variety in road rendering
- **Accessibility**: ARIA labels, keyboard navigation improvements

### Low Priority

- **Animations**: More dramatic effects for generation
- **Sound Effects**: Optional audio (keep it optional!)
- **Themes**: Alternative color schemes while maintaining aesthetic

## Testing

Before submitting:

- [ ] Code compiles without errors
- [ ] No TypeScript errors
- [ ] ESLint passes
- [ ] All features work in Chrome, Firefox, Safari
- [ ] Print functionality works correctly
- [ ] No console errors

## Pull Request Guidelines

1. **One feature per PR**: Keep changes focused
2. **Descriptive titles**: "Add mobile sidebar collapse" not "Fix stuff"
3. **Test your changes**: Include testing notes in PR description
4. **Respect the aesthetic**: Keep the MÖRK BORG visual identity
5. **Update documentation**: If you add features, document them

## File Organization

- **Components**: `src/components/` - React components
- **Data**: `src/data/` - Game tables and content
- **Utils**: `src/utils/` - Logic and helpers
- **Types**: `src/types/` - TypeScript interfaces
- **Styles**: `src/index.css` - Global styles

## Questions?

Open an issue for discussion before starting major features.
